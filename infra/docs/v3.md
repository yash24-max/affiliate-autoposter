# Infra V3 — Delta (Multi-Platform + AI)

This document covers only what changes or is added in V3. All V1 and V2 infra configuration remains. See [v1.md](v1.md) for the full baseline.

---

## New Infrastructure Components

### Kafka (Managed)
- Use a managed Kafka service (Confluent Cloud, MSK, or Upstash Kafka)
- Topics: `product.ready`, `post.result`, `click.event`
- Retention: 7 days per topic
- Environment variables to add:
  ```
  KAFKA_BOOTSTRAP_SERVERS    Managed Kafka broker addresses
  KAFKA_API_KEY              Kafka auth key
  KAFKA_API_SECRET           Kafka auth secret
  ```

### ClickHouse Cluster
- Use managed ClickHouse (ClickHouse Cloud or self-managed on EC2)
- Single analytics node for V3; scale out later
- Environment variables to add:
  ```
  CLICKHOUSE_HOST
  CLICKHOUSE_PORT
  CLICKHOUSE_USER
  CLICKHOUSE_PASSWORD
  CLICKHOUSE_DATABASE
  ```

### Publisher Workers as Separate Services
Each platform publisher (Instagram, Twitter/X, WhatsApp, Facebook) runs as its own deployable service and Kafka consumer. Each has its own container and CI/CD pipeline stage.

---

## Deployment Target Migration

V3 moves from Railway/Render to **AWS EC2 / ECS** to support:
- More containers (Kafka consumers, ClickHouse, AI Service)
- Better network isolation (VPC, security groups)
- Managed services alignment (RDS in V4, MSK Kafka)

ECS Fargate is the recommended V3 target: managed container orchestration without Kubernetes overhead.
