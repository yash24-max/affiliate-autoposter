# Backend V4 — Delta (Full SaaS Scale)

This document covers only what changes or is added in V4. All prior version architecture remains. See [v1.md](v1.md) for the full baseline.

---

## New Services

### Billing Service (Port 8091)
- Manages subscription plans (FREE, PRO, AGENCY)
- Integrates with Razorpay (India) and Stripe (global) for recurring billing
- Webhooks from payment providers update plan status
- Plan enforcement signals propagated to all services via user_profiles.plan field
- Endpoints: plan status, upgrade/downgrade, payment session creation, webhook handler

### Multi-Tenancy Support
- Agency accounts can manage multiple client sub-accounts
- Each sub-account is a full user with isolated data
- Agency owner can view/manage all sub-accounts
- Team members with role-based access (VIEWER, EDITOR, ADMIN)

### White-Label Service
- Stores per-agency branding config (logo, colors, custom domain)
- Applied at API Gateway level for custom domain routing
- Endpoints: get/update branding config

## Data Model Additions

### user_config.user_profiles additions
```sql
agency_owner_id UUID  -- NULL for regular users; set for sub-accounts managed by an agency
stripe_customer_id VARCHAR
razorpay_subscription_id VARCHAR
```

### New table: billing.subscriptions
```sql
billing.subscriptions
  id UUID PK, user_id UUID NOT NULL,
  plan VARCHAR NOT NULL,  (FREE, PRO, AGENCY)
  status VARCHAR,  (ACTIVE, CANCELLED, PAST_DUE)
  provider VARCHAR,  (RAZORPAY, STRIPE)
  external_subscription_id VARCHAR,
  current_period_start TIMESTAMP, current_period_end TIMESTAMP,
  created_at TIMESTAMP, updated_at TIMESTAMP
```

### New table: agency.team_members
```sql
agency.team_members
  id UUID PK, agency_user_id UUID NOT NULL,
  member_user_id UUID NOT NULL,
  role VARCHAR,  (VIEWER, EDITOR, ADMIN)
  created_at TIMESTAMP
```
