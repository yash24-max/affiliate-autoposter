# Backend V2 — Delta (Visual Layer + Pinterest)

This document covers only what changes or is added in V2. All V1 architecture, APIs, and data model remain. See [v1.md](v1.md) for the full baseline.

---

## New Services

### Image Service (Port 8087)
- Receives product payload from Fetcher Service
- Selects the user's preferred template
- Renders HTML/CSS template using Playwright (headless Chromium sidecar)
- Exports PNG and uploads to Cloudinary
- Returns Cloudinary CDN URL to Pusher Service

### Pinterest Publisher (within Pusher Service or new Port 8088)
- Receives product payload + image URL from Fetcher
- Builds Pinterest Pin: image, title, description, affiliate link
- Posts to user's Pinterest board via Pinterest API v5
- Records post result in `post_events.posts` with `platform = 'PINTEREST'`
- Retry logic: same pattern as Telegram publisher (3 attempts, exponential backoff)

---

## New Tech Introduced

| Technology | Purpose |
|-----------|---------|
| Playwright (Java/Node sidecar) | Headless browser for HTML → PNG rendering |
| Cloudinary SDK | Image upload and CDN URL generation |
| Pinterest API v5 | Publishing pins with affiliate links |
| Feign Client | Replaces RestTemplate for cleaner external API calls |

## Data Model Additions

### user_config schema additions
```sql
user_config.pinterest_credentials
  id UUID PK, user_id UUID UNIQUE NOT NULL,
  access_token_enc VARCHAR NOT NULL  (AES-256-GCM encrypted),
  board_id VARCHAR NOT NULL,
  is_active BOOLEAN, last_verified_at TIMESTAMP,
  created_at TIMESTAMP, updated_at TIMESTAMP

user_config.user_profiles additions:
  preferred_template_id VARCHAR  -- references template catalog
```
