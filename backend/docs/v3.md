# Backend V3 — Delta (Multi-Platform + AI)

This document covers only what changes or is added in V3. All prior version architecture remains. See [v1.md](v1.md) for the full baseline.

---

## New Services

### AI Service (Port 8089)
- Receives candidate product list from Fetcher Service
- Calls OpenAI / Gemini API via Spring AI to score products by relevance (season, trends, user history)
- Returns ranked product list back to Fetcher for selection
- Can be toggled per user via user preferences

### Platform Publisher Workers (separate deployables)
Each platform gets its own publisher process that consumes from a Kafka topic:
- **Instagram Publisher**: Meta Business API
- **Twitter/X Publisher**: Twitter API v2
- **WhatsApp Publisher**: WhatsApp Business API
- **Facebook Publisher**: Meta Graph API

### Analytics Service (Port 8090)
- Consumes click events from Kafka topic
- Writes to ClickHouse for high-performance aggregation
- Exposes `GET /api/analytics/clicks` and summary endpoints
- Owns `analytics` ClickHouse schema

### URL Shortener Module (within Pusher or standalone)
- Generates short redirect URLs for all affiliate links
- Tracks clicks: platform, country (from IP), device (from UA)
- Emits click events to Kafka topic

---

## Architecture Change: Synchronous → Event-Driven

V1/V2: `Fetcher → Pusher (REST)`
V3: `Fetcher → Kafka topic "product.ready" → platform-specific consumers`

Each publisher subscribes independently. Adding a new platform = new Kafka consumer, no changes to Fetcher.

```
Fetcher publishes → Kafka: product.ready
  ├── Telegram Publisher consumes
  ├── Pinterest Publisher consumes
  ├── Instagram Publisher consumes
  ├── Twitter Publisher consumes
  └── WhatsApp Publisher consumes
Each publisher emits → Kafka: post.result
  └── Analytics/Dashboard consumers
```

## Data Model Additions

### New ClickHouse schema: analytics
```sql
analytics.click_events
  id UUID, post_id UUID, user_id UUID,
  platform VARCHAR, short_url VARCHAR,
  clicked_at TIMESTAMP, country_code VARCHAR,
  device_type VARCHAR, ip_hash VARCHAR (hashed for privacy)
```

### New Kafka topics
```
product.ready       Fetcher → platform publishers
post.result         publishers → dashboard/analytics
click.event         URL shortener → analytics service
```

### user_config.user_profiles additions
```sql
ai_picker_enabled BOOLEAN DEFAULT FALSE
```
