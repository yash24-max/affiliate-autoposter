# Infra V2 — Delta (Visual Layer + Pinterest)

This document covers only what changes or is added in V2. All V1 infra configuration remains. See [v1.md](v1.md) for the full baseline.

---

## New: Playwright Sidecar Container

The Image Service requires a Playwright (headless Chromium) process to render HTML → PNG. This runs as a sidecar container alongside the Image Service.

**Docker Compose addition for local dev:**
```yaml
playwright-sidecar:
  image: mcr.microsoft.com/playwright:v1.40.0-jammy
  # or custom image with Playwright + Node server
  environment:
    - PORT=3001
  networks:
    - autoposter-net
```

**Environment variable additions:**
```
CLOUDINARY_CLOUD_NAME   Cloudinary account name
CLOUDINARY_API_KEY      Cloudinary API key
CLOUDINARY_API_SECRET   Cloudinary API secret (store encrypted)
PLAYWRIGHT_SERVICE_URL  Internal URL of the Playwright sidecar
```

---

## Cloudinary Integration

- Images rendered by Playwright are uploaded to Cloudinary via SDK
- Cloudinary URLs served directly in posts (no proxy needed)
- Backup: daily export of image inventory from Cloudinary to object storage

---

## No Deployment Target Change

V2 remains on Railway or Render. The Playwright sidecar and Image Service add two new containers to the compose file but do not require a new deployment platform.

---

## CI/CD Addition

Add `image-service` and `playwright-sidecar` to the Docker build and deploy pipeline stages.
