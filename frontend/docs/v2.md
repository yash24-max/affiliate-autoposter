# Frontend V2 — Delta (Visual Layer + Pinterest)

This document covers only what changes or is added in V2. All V1 features remain. See [v1.md](v1.md) for the full baseline.

---

## New Screens

### Template Picker
- Gallery view of available product card templates (5–10 designs)
- Preview cards render a sample product in each template style
- User selects preferred template → saved via `PUT /api/user/template-preference`
- Accessible from settings and from the schedule page

### Pinterest Settings
- New section in settings: Pinterest account connection
- Input: Pinterest access token and board ID
- Test connection button → `POST /api/pinterest-config/test`
- Toggle to enable Pinterest alongside Telegram

---

## New API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/user/template-preference` | Get selected template |
| PUT | `/api/user/template-preference` | Set preferred template |
| GET | `/api/pinterest-config` | Get Pinterest config (token never returned) |
| PUT | `/api/pinterest-config` | Save Pinterest credentials |
| POST | `/api/pinterest-config/test` | Test Pinterest connection |

---

## No New Frontend Dependencies

V2 requires no additional npm packages. The Playwright image rendering happens entirely on the backend (sidecar). Cloudinary image URLs are served as standard `<img src>` tags.

---

## UX Notes

- Generated images load from Cloudinary CDN — no lazy-loading issues expected
- Template picker shows loading skeleton while templates load
- Pinterest config section hidden if user is on FREE plan (feature gated by plan field in auth token)
